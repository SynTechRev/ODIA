# Extraction Directory

This directory stores extracted text content from audited documents. Text files are generated by OCR or PDF text extraction processes.

## File Naming Convention

Extracted text files should be named: `{document_id}.txt`

Example: `DOC123.txt`, `FDA_2024_001.txt`

## Usage

Extract text using the OCR sample script:

```bash
python scripts/ocr_sample.py --input /external/storage/file.pdf --page 1 --out manifests/DOC123.json
```

This will:
1. Extract text from the specified PDF page(s)
2. Save text to `extraction/{document_id}.txt`
3. Update the manifest with extraction metadata

## Text Format

- Plain text UTF-8 encoding
- Preserves line breaks and paragraph structure where possible
- May include OCR artifacts or recognition errors (check extraction confidence in manifest)

## Security Warning

⚠️ **Do NOT commit extraction files containing PII or sensitive information without proper redaction.**

Review extracted text before committing to the repository. Use the redaction configuration in `config/defaults.yaml` to enable PII detection placeholders.

## Storage Considerations

Extracted text files can be large. Consider:
- Adding `extraction/*.txt` to `.gitignore` if files exceed repository size limits
- Storing large extraction files alongside external PDFs
- Using compression for archival storage

## Associated Manifests

Each extraction file should have a corresponding manifest in the `manifests/` directory with:
- `extraction.text_present: true`
- `extraction.extracted_text_path` pointing to this file
- `extraction.ocr_used` indicating whether OCR was required
- `extraction.extraction_confidence` score (0-1)
